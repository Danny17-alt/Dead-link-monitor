# DeadLink Monitor — Code Workflow

## Repository
`Danny17-alt/Dead-link-monitor` on GitHub

## Branch Strategy
- `main` — production-ready code
- `feature/*` — feature branches for new work

## Process
1. Create a feature branch from `main`
2. Commit and push changes
3. Open a pull request to `main`
4. Lead reviews and merges using `gh pr merge`

## Team Members
- `agent-lead` — lead, reviews PRs
- `agent-backend-engineer` — backend API work
- `agent-frontend-engineer` — frontend dashboard work
- `agent-scanner-engineer` — scanner engine work
